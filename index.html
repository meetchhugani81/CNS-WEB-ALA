Digital Signature System
Public-Key Cryptography RSA-2048 SHA-256 Sign & Verify

A Digital Signature provides authentication, integrity, and non-repudiation. The message is first hashed with SHA-256, then the hash is encrypted with the sender's private key to produce the signature. The receiver decrypts with the public key and compares against a freshly computed hash to verify authenticity.

Message → SHA-256 → Hash → Sign (Pvt Key) → Signature

Hash Function Comparison
SHA-1 SHA-256 SHA-512 Avalanche Effect

Cryptographic hash functions map arbitrary input to a fixed-length digest. The Avalanche Effect ensures that a single-character change causes ~50% of output bits to flip — making it impossible to predict hash outputs. SHA-1 = 160 bits, SHA-256 = 256 bits, SHA-512 = 512 bits.

HMAC — Message Auth Code
HMAC-SHA256 Shared Secret Key Sender & Receiver Sim

HMAC (Hash-based Message Authentication Code) uses a shared secret key combined with a hash function to produce an authentication tag. Both sender and receiver share the same key. The receiver recomputes the HMAC and checks it against the received tag to verify integrity and authenticity.

Sender → Msg + Key → HMAC-SHA256 → MAC Tag → Receiver

Note
⚠ Tamper Detection Explanation

When a message is tampered with, even a single character change causes the HMAC to differ completely due to the avalanche effect in SHA-256. Since the attacker does not know the secret key, they cannot forge a valid HMAC for the modified message. This is the core security property of HMAC — it provides both integrity (message unchanged) and authenticity (only key-holders can generate valid tags).
